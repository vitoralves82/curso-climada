Prática 1: Instalação e Primeiros Passos com CLIMADA

Nesta prática, você irá configurar seu ambiente, realizar os primeiros testes com o CLIMADA e explorar sua API básica. Ao final, você terá um ambiente funcional e familiaridade com os componentes essenciais.

Conteúdo da Prática

1. Instalação do Ambiente CLIMADA 2. Configuração do Jupyter Notebook/Lab 3. Primeiro Contato com a API CLIMADA 4. Exercícios Práticos

Pré-requisitos

  • Conhecimentos básicos de linha de comando (terminal/prompt)
  • Familiaridade com Python (variáveis, funções, importações)
  • Computador com acesso à internet e pelo menos 8GB de RAM (16GB+ recomendado)
  • Instalador Mamba ou Conda (Miniconda/Anaconda)

1. Instalação do Ambiente CLIMADA

Instalaremos o CLIMADA usando Mamba (recomendado) em um ambiente virtual dedicado.

1.1. Criação do Ambiente Virtual

Abra seu terminal (Anaconda Prompt no Windows, Terminal no macOS/Linux) e crie o ambiente: 

mamba create -n climada_env python=3.9 -c conda-forge -y

# O '-y' confirma automaticamente as perguntas.

1.2. Ativação do Ambiente

Ative o ambiente recém-criado: 

mamba activate climada_env

# Seu prompt deve mudar para indicar "(climada_env)".

1.3. Instalação do CLIMADA Core

Com o ambiente ativo, instale o pacote principal do CLIMADA: 

mamba install -c conda-forge climada -y

Atenção: A instalação pode demorar alguns minutos. Se encontrar erros de dependência, tente: mamba install -c conda-forge climada --strict-channel-priority -y

1.4. Verificação da Instalação

Verifique a instalação executando no terminal: 

python -c "import climada; print(f'CLIMADA versão: {climada.__version__}')"

Você deve ver a versão instalada do CLIMADA.

Opcional: CLIMADA Petals

Para funcionalidades extras (módulos específicos de perigo/exposição), instale o Petals:

mamba install -c conda-forge climada_petals -y

2. Configuração do Jupyter Notebook/Lab

Usaremos o JupyterLab (uma interface mais moderna) para nossas análises interativas.

2.1. Instalação do JupyterLab

No ambiente climada_env ativado, instale o JupyterLab: 

mamba install -c conda-forge jupyterlab -y

2.2. Iniciando o JupyterLab

Navegue no terminal até a pasta onde deseja salvar seus notebooks (ex: Documentos/CursoCLIMADA) e inicie o JupyterLab: 

jupyter lab

Seu navegador padrão abrirá com a interface do JupyterLab.

2.3. Criando seu Primeiro Notebook

No lançador do JupyterLab (aba "Launcher"), clique em "Python 3 (ipykernel)" sob "Notebook". Renomeie o arquivo (clicando com o botão direito na aba ou no nome "Untitled.ipynb" no explorador de arquivos) para Pratica1_CLIMADA.ipynb.

2.4. Teste Rápido no Notebook

Na primeira célula do notebook, digite e execute (Shift+Enter) o código de verificação novamente: 

import climada
print(f"CLIMADA versão: {climada.__version__}")

A versão deve ser exibida abaixo da célula, confirmando que o kernel do notebook está usando o ambiente correto.

Dica: Familiarize-se com a interface do JupyterLab: explorador de arquivos à esquerda, área principal para notebooks/editores, e abas na parte superior.

3. Primeiro Contato com a API CLIMADA

Vamos explorar os componentes básicos da API do CLIMADA no seu notebook Pratica1_CLIMADA.ipynb. Execute cada bloco de código em células separadas.

3.1. Importações Essenciais

Comece com as importações necessárias: 

import numpy as np
import matplotlib.pyplot as plt
from pathlib import Path # Para gerenciar caminhos de arquivos

# Componentes CLIMADA
from climada.hazard import Hazard, Centroids, TCTracks, TropCyclone
from climada.entity import Exposures, Entity, ImpactFuncSet
from climada.entity.impact_funcs.trop_cyclone import ImpfTropCyclone # Função de impacto específica
from climada.engine import Impact

# Configuração para gráficos inline
%matplotlib inline
# Define diretório de dados (opcional, mas útil)
DATA_DIR = Path.home() / "climada_data"
DATA_DIR.mkdir(parents=True, exist_ok=True)
print(f"Diretório de dados: {DATA_DIR}")

3.2. Centroids: Pontos de Cálculo

Crie centroids para uma região (ex: Filipinas Central): 

# Definir limites para as Filipinas Centrais (aproximado)
lat_min, lat_max = 8, 15
lon_min, lon_max = 120, 127
resolution_deg = 0.1 # Resolução em graus

# Criar centroids
centroids_phl = Centroids.from_pnt_bounds((lon_min, lat_min, lon_max, lat_max), res=resolution_deg)
centroids_phl.check() # Verifica a estrutura dos centroids

# Visualizar
fig, ax = plt.subplots(1, 1, figsize=(8, 8))
centroids_phl.plot(ax=ax)
ax.set_title(f"{len(centroids_phl.lat)} Centroids nas Filipinas Centrais")
plt.show()

Centroids representam os locais onde calcularemos a intensidade do perigo e o impacto.

3.3. TCTracks: Trajetórias de Ciclones

Carregue dados de um ciclone histórico (ex: Tufão Haiyan/Yolanda 2013): 

# Carregar trajetória do Tufão Haiyan (ID: 2013306N06150) do IBTrACS
# O CLIMADA tentará baixar os dados se não existirem localmente
try:
    tracks_haiyan = TCTracks.from_ibtracs_netcdf(storm_id='2013306N06150',
                                                 provider='usa', # Fonte dos dados
                                                 data_dir=DATA_DIR / "ibtracs") # Onde salvar/procurar
    tracks_haiyan.equal_timestep(time_step_h=1.0) # Interpolar para passos de 1 hora

    # Visualizar trajetória
    fig, ax = plt.subplots(1, 1, figsize=(10, 6))
    tracks_haiyan.plot(ax=ax)
    centroids_phl.plot(ax=ax, color='gray', alpha=0.5, markersize=1) # Plotar centroids também
    ax.set_title(f"Trajetória do Tufão Haiyan (Yolanda) - {tracks_haiyan.data[0].attrs['name']}")
    plt.show()

    # Informações básicas
    print(tracks_haiyan.data[0]) # Mostra detalhes da trajetória

except Exception as e:
    print(f"Erro ao carregar dados do IBTrACS: {e}")
    print("Verifique sua conexão ou o diretório de dados.")

Download de Dados: A primeira execução pode demorar enquanto o CLIMADA baixa os dados necessários do IBTrACS. Certifique-se de estar conectado à internet.

3.4. TropCyclone (Hazard): Modelando o Perigo

Use a trajetória para modelar o campo de vento do ciclone nos centroids: 

# Criar o objeto Hazard (perigo)
tc_haiyan_hazard = TropCyclone.from_tracks(tracks_haiyan, centroids=centroids_phl)
tc_haiyan_hazard.check()

# Visualizar a intensidade máxima do vento nos centroids
fig, ax = plt.subplots(1, 1, figsize=(10, 8))
tc_haiyan_hazard.plot_intensity(event=0, ax=ax) # event=0 pega o primeiro (e único) evento carregado
ax.set_title(f"Intensidade Máxima do Vento (m/s) - Tufão Haiyan nos Centroids")
plt.show()

O objeto Hazard contém a intensidade do perigo (aqui, velocidade do vento) em cada centroid para cada evento.

3.5. Exposures: Modelando Ativos

Use o LitPop para criar uma exposição econômica para a região: 

from climada.entity.exposures.litpop import LitPop

try:
    # Criar exposição LitPop para as Filipinas (pode demorar na 1ª vez)
    # Usaremos uma resolução menor (300 arc-segundos ~ 9km) para acelerar
    exp_phl = LitPop.from_countries('PHL', res_arcsec=300, data_dir=DATA_DIR / "litpop")
    exp_phl.check()

    # Associar a função de impacto padrão para TC (usaremos ID 1)
    exp_phl.gdf['impf_TC'] = 1

    # Filtrar exposição para a área dos centroids (opcional, mas melhora performance)
    exp_phl.crop(centroids_phl.buffer) # Corta usando um buffer ao redor dos centroids

    # Visualizar a exposição
    fig, ax = plt.subplots(1, 1, figsize=(10, 8))
    exp_phl.plot_hexbin(ax=ax, pop_name=False, buffer=0.1) # buffer para margem
    ax.set_title(f"Exposição LitPop (Valor em USD) - Filipinas Centrais (Res: 300as)")
    plt.show()

    print(f"Total de pontos de exposição: {len(exp_phl.gdf)}")
    print(f"Valor total exposto: {exp_phl.gdf.value.sum():,.0f} USD")

except Exception as e:
    print(f"Erro ao criar LitPop: {e}")
    print("Pode ser necessário baixar dados do GPW ou Nightlight. Verifique a documentação do LitPop.")

O objeto Exposures contém a localização e o valor dos ativos expostos.

3.6. Impact Functions: Modelando Vulnerabilidade

Defina como a intensidade do vento se traduz em dano: 

# Usar uma função de impacto padrão para TC (baseada em Emanuel 2011)
impf_tc_phl = ImpfTropCyclone.from_emanuel_asia() # Função calibrada para Ásia
impf_tc_phl.id = 1 # Definir o ID que usamos na exposição (impf_TC = 1)
impf_tc_phl.check()

# Criar o conjunto de funções de impacto
impact_func_set = ImpactFuncSet([impf_tc_phl])

# Visualizar a curva
fig, ax = plt.subplots(1, 1, figsize=(8, 5))
impact_func_set.plot(axis=ax)
ax.set_title("Função de Impacto TC (Emanuel Ásia)")
ax.set_xlabel("Velocidade do Vento (m/s)")
ax.set_ylabel("Fração Média de Dano (MDR)")
plt.show()

A ImpactFuncSet contém as curvas que definem a vulnerabilidade dos ativos ao perigo.

3.7. Engine (Impact): Calculando o Impacto

Combine tudo para calcular o impacto financeiro: 

# Calcular o impacto do evento Haiyan
impact_haiyan = Impact()
impact_haiyan.calc(exp_phl, impact_func_set, tc_haiyan_hazard)
impact_haiyan.check()

# Impacto total do evento
total_impact_usd = impact_haiyan.imp_mat.sum()
print(f"Impacto total estimado do Tufão Haiyan nesta região: {total_impact_usd:,.0f} USD")

# Visualizar o impacto espacialmente
fig, ax = plt.subplots(1, 1, figsize=(10, 8))
impact_haiyan.plot_spatial_impact(event=0, haz_id=tc_haiyan_hazard.event_id[0], ax=ax)
ax.set_title("Impacto Espacial Estimado - Tufão Haiyan (USD)")
plt.show()

# Calcular métricas de risco (Ex: Dano Anual Esperado - requer mais eventos)
# aai = impact_haiyan.aai_agg # Só faz sentido com múltiplos eventos
# print(f"Dano Anual Esperado (AAI) agregado: {aai:,.0f} USD")
# impact_haiyan.plot_rp_intensity() # Curva de período de retorno

O objeto Impact realiza o cálculo principal, combinando perigo, exposição e vulnerabilidade para estimar os danos. Para métricas de risco probabilísticas como AAI, precisaríamos de um conjunto maior de eventos de perigo.
