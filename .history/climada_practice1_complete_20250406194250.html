Prática 1: Instalação e Primeiros Passos com CLIMADA

Nesta prática, você irá configurar seu ambiente de trabalho para utilizar o CLIMADA, realizar os primeiros testes e explorar a API básica da plataforma. Ao final desta prática, você terá um ambiente funcional e estará familiarizado com os componentes principais do CLIMADA.

Conteúdo da Prática

1. Instalação do Ambiente CLIMADA 2. Configuração do Jupyter Notebook 3. Primeiro Contato com a API CLIMADA 4. Exercícios Práticos

Pré-requisitos

  • Conhecimentos básicos de linha de comando
  • Familiaridade com Python (variáveis, funções, importações)
  • Computador com acesso à internet e pelo menos 8GB de RAM

1. Instalação do Ambiente CLIMADA

O CLIMADA é distribuído como um pacote Python e pode ser instalado usando o gerenciador de pacotes conda ou mamba. Recomendamos fortemente o uso do mamba, que é mais rápido e eficiente na resolução de dependências.

1.1. Instalação do Conda/Mamba

Se você ainda não tem o Anaconda ou Miniconda instalado, você pode baixá-lo e instalá-lo seguindo as instruções em docs.anaconda.com/anaconda/install/.

Após instalar o Conda, instale o Mamba, que é uma reimplementação mais rápida do Conda: 

conda install -c conda-forge mamba

1.2. Criação de um Ambiente Virtual para o CLIMADA

É altamente recomendável criar um ambiente virtual dedicado para o CLIMADA, a fim de evitar conflitos com outras bibliotecas: 

mamba create -n climada_env python=3.9 -c conda-forge

Após a criação, ative o ambiente: 

# No Windows:
activate climada_env

# No macOS e Linux:
source activate climada_env

1.3. Instalação do CLIMADA

Agora, instale o CLIMADA usando o mamba: 

mamba install -c conda-forge climada

Atenção: A instalação pode levar alguns minutos, pois o CLIMADA tem várias dependências. Se ocorrerem erros de resolução de dependências, tente o seguinte comando:

mamba install -c conda-forge climada --strict-channel-priority

1.4. Verificação da Instalação

Para verificar se o CLIMADA foi instalado corretamente, execute o seguinte comando no terminal: 

python -c "import climada; print(climada.__version__)"

Você deve ver a versão do CLIMADA, indicando que a instalação foi bem-sucedida.

Dica: Se você planeja trabalhar com funções avançadas, como modelagem de perigos específicos não incluídos no core, você também pode instalar o pacote climada_petals:

mamba install -c conda-forge climada_petals

2. Configuração do Jupyter Notebook

O Jupyter Notebook é uma ferramenta excelente para trabalhar com o CLIMADA, pois permite combinar código, visualizações e explicações em um único documento interativo.

2.1. Instalação do Jupyter Notebook

Se você ainda não tem o Jupyter Notebook instalado no seu ambiente CLIMADA, instale-o usando: 

mamba install -c conda-forge jupyter notebook

2.2. Iniciar o Jupyter Notebook

Com o ambiente climada_env ativado, inicie o Jupyter Notebook: 

jupyter notebook

Isso abrirá uma nova janela ou guia no seu navegador com a interface do Jupyter Notebook.

2.3. Criar um Novo Notebook

Na interface do Jupyter, clique em "New" e selecione "Python 3" para criar um novo notebook. Renomeie o notebook para "CLIMADA_Teste_Inicial.ipynb" clicando no título "Untitled" e digitando o novo nome.

2.4. Verificar se o CLIMADA está Acessível

Na primeira célula do notebook, digite e execute o seguinte código para verificar se o CLIMADA está acessível: 

import climada
print(f"CLIMADA versão: {climada.__version__}")

A saída deve mostrar a versão do CLIMADA, confirmando que tudo está configurado corretamente.

Dica: Jupyter Notebook tem muitos atalhos de teclado úteis. Pressione Shift+Enter para executar a célula atual e avançar para a próxima, ou Ctrl+Enter para executar a célula atual sem avançar.

Próxima Seção

3. Primeiro Contato com a API CLIMADA

Nesta seção, vamos explorar os principais componentes da API do CLIMADA: Hazard, Exposures, Impact Functions e Engine. Você aprenderá como importar e utilizar cada um desses componentes.

3.1. Importação dos Componentes Principais

Comece importando os principais módulos do CLIMADA: 

# Importação dos principais componentes do CLIMADA
import numpy as np
import matplotlib.pyplot as plt

# Módulos do CLIMADA
from climada.hazard import Hazard, TCTracks, TropCyclone, Centroids
from climada.entity import Exposures, Entity, ImpactFuncSet
from climada.entity.impact_funcs import ImpfTropCyclone
from climada.engine import ImpactCalc

# Configuração para visualização de gráficos inline no notebook
%matplotlib inline

3.2. Explorando a Classe Centroids

Vamos criar um objeto Centroids simples para entender como funciona: 

# Criando um conjunto de centroids para uma área específica
min_lat, max_lat = 17.5, 19.0  # Exemplo para Porto Rico
min_lon, max_lon = -68.0, -65.0
resolution = 0.1  # resolução em graus (aproximadamente 10km no equador)

# Criação dos centroids
cent = Centroids.from_pnt_bounds((min_lon, min_lat, max_lon, max_lat), res=resolution)

# Verificação dos centroids
cent.check()

# Visualização dos centroids
plt.figure(figsize=(10, 6))
cent.plot()
plt.title('Centroids para Porto Rico')
plt.show()

Este código cria uma grade de pontos (centroids) que cobrem Porto Rico. Os centroids são pontos geográficos onde o impacto do perigo será calculado.

3.3. Explorando a Classe TCTracks

Agora, vamos explorar como carregar dados de ciclones tropicais: 

# Carregando um evento histórico específico - Furacão Irma de 2017
# Este código fará o download dos dados do IBTrACS a primeira vez que for executado
try:
    print("Baixando dados do IBTrACS para o Furacão Irma (2017)...")
    tr_irma = TCTracks.from_ibtracs_netcdf(provider='usa', storm_id='2017242N16333')
    
    # Visualização da trajetória
    plt.figure(figsize=(12, 8))
    ax = tr_irma.plot()
    ax.set_title('Trajetória do Furacão Irma (2017)')
    plt.show()
    
    # Exibindo as informações básicas do track
    track = tr_irma.data[0]
    print(f"Nome do Evento: {track.attrs['name']}")
    print(f"Categoria Máxima: {track.attrs['category']}")
    print(f"Data de Início: {track.time.values[0]}")
    print(f"Data de Término: {track.time.values[-1]}")
except Exception as e:
    print(f"Erro ao carregar os dados: {e}")
    print("Você pode precisar baixar manualmente o arquivo IBTrACS.ALL.v04r00.nc")
    print("Visite: https://www.ncei.noaa.gov/data/international-best-track-archive-for-climate-stewardship-ibtracs/v04r00/access/netcdf/")

Este código baixa e visualiza a trajetória do Furacão Irma de 2017. O CLIMADA usa estes dados de trajetória para modelar os campos de vento associados ao ciclone.

3.4. Explorando a Classe TropCyclone

Vamos criar um objeto TropCyclone usando a trajetória do Irma: 

# Criando um objeto TropCyclone a partir da trajetória do Irma
try:
    # Primeiro, vamos adicionar passos de tempo mais frequentes para uma melhor resolução
    tr_irma.equal_timestep(time_step_h=0.5)
    
    # Criando o objeto TropCyclone com os centroids definidos anteriormente
    tc_hazard = TropCyclone.from_tracks(tr_irma, centroids=cent)
    
    # Verificação do objeto hazard
    tc_hazard.check()
    
    # Visualização da intensidade do ciclone
    plt.figure(figsize=(12, 8))
    tc_hazard.plot_intensity(event='2017242N16333')
    plt.title('Intensidade do Furacão Irma (2017)')
    plt.show()
except Exception as e:
    print(f"Erro ao processar o ciclone: {e}")

Este código cria um objeto TropCyclone que modela o campo de vento do Furacão Irma nos centroids que definimos para Porto Rico.

3.5. Explorando a Classe Exposures

Agora, vamos criar uma exposição simples para Porto Rico usando o módulo LitPop: 

# Importando o módulo LitPop para criar exposições econômicas
from climada.entity.exposures import LitPop

try:
    # Criando uma exposição para Porto Rico
    print("Gerando exposição para Porto Rico...")
    exp_pr = LitPop.from_countries('Puerto Rico', res_arcsec=120)
    
    # Configurando a função de impacto a ser usada para ciclones tropicais
    exp_pr.gdf['impf_TC'] = 1
    
    # Visualização da exposição
    plt.figure(figsize=(12, 8))
    exp_pr.plot_hexbin(pop_name=True)
    plt.title('Exposição Econômica em Porto Rico')
    plt.show()
except Exception as e:
    print(f"Erro ao criar exposição: {e}")
    print("Nota: Pode ser necessário baixar dados adicionais. Siga as instruções no console.")

O código acima cria uma exposição econômica para Porto Rico usando o módulo LitPop, que combina dados de luzes noturnas e população para estimar valores econômicos expostos.

3.6. Explorando as Funções de Impacto

Vamos criar uma função de impacto para ciclones tropicais: 

# Criando uma função de impacto para ciclones tropicais
imp_func = ImpfTropCyclone.from_emanuel_usa()

# Criando um conjunto de funções de impacto
impact_funcs = ImpactFuncSet([imp_func])

# Visualizando a função de impacto
plt.figure(figsize=(10, 6))
imp_func.plot()
plt.title('Função de Impacto para Ciclones Tropicais (Emanuel)')
plt.show()

# Exibindo informações sobre a função de impacto
print(f"ID da Função: {imp_func.id}")
print(f"Nome: {imp_func.name}")
print(f"Tipo de Perigo: {imp_func.haz_type}")

Este código cria uma função de impacto baseada no modelo de Emanuel para ciclones tropicais, que relaciona a velocidade do vento ao dano esperado em estruturas.

3.7. Calculando o Impacto

Finalmente, vamos calcular o impacto combinando o perigo, a exposição e a função de impacto: 

try:
    # Calculando o impacto
    impact = ImpactCalc(exp_pr, impact_funcs, tc_hazard).impact()
    
    # Verificando o impacto total
    print(f"Impacto Total: {impact.aai_agg:.2f} USD")
    
    # Visualizando a curva de frequência de impacto
    plt.figure(figsize=(10, 6))
    impact.calc_freq_curve().plot()
    plt.title('Curva de Frequência de Excedência de Impacto')
    plt.xlabel('Impacto (USD)')
    plt.ylabel('Frequência (1/ano)')
    plt.show()
    
    # Visualizando o impacto espacial
    plt.figure(figsize=(12, 8))
    impact.plot_basemap_eai_exposure(buffer=0.1)
    plt.title('Impacto Anual Esperado por Exposição')
    plt.show()
except Exception as e:
    print(f"Erro ao calcular impacto: {e}")

Este código calcula e visualiza o impacto do Furacão Irma em Porto Rico, combinando o perigo modelado, a exposição econômica e a função de impacto.

Nota: O cálculo de impacto pode levar algum tempo, dependendo da resolução dos centroids e da extensão da área de estudo.

Exercício 2: Explorando Cenários de Mudanças Climáticas

O CLIMADA permite simular os efeitos das mudanças climáticas nos ciclones tropicais. Vamos explorar essa funcionalidade: 

# EXERCÍCIO 2: Aplicar um cenário de mudança climática ao Furacão Irma

# Seu código aqui:
# 1. Carregue novamente os dados do Furacão Irma
# 2. Crie o objeto TropCyclone
# 3. Aplique um cenário de mudança climática para 2050 usando o método apply_climate_scenario_knu
# 4. Compare a intensidade do ciclone no cenário atual e futuro

# Dica: Use o método apply_climate_scenario_knu com os parâmetros ref_year e rcp_scenario
# Exemplo: tc_hazard_future = tc_hazard.apply_climate_scenario_knu(ref_year=2050, rcp_scenario=85)

Sobre a função apply_climate_scenario_knu

Esta função implementa mudanças na intensidade e frequência dos ciclones tropicais com base no estudo "Global projections of intense tropical cyclone activity for the late twenty-first century from dynamical downscaling of CMIP5/RCP4.5 scenarios" de Knutson et al. (2015).

Parâmetros principais:

  • ref_year: ano de referência para as projeções (ex: 2050)
  • rcp_scenario: cenário de emissões (45 para RCP 4.5 ou 85 para RCP 8.5)

from climada.entity.impact_funcs import ImpactFunc

# Seu código aqui:
# 1. Crie um objeto ImpactFunc para o tipo de perigo 'TC'
# 2. Defina a intensidade (velocidade do vento) e os valores de MDD e PAA para embarcações offshore
# 3. Visualize a função de impacto
# 4. Compare com a função de Emanuel

# Dica: Embarcações offshore geralmente começam a sofrer danos com ventos mais fortes,
# mas a curva de dano pode ser mais íngreme após um certo limiar.

# Exemplo de como iniciar:
impact_func_offshore = ImpactFunc()
impact_func_offshore.id = 2
impact_func_offshore.name = "Embarcações Offshore"
impact_func_offshore.haz_type = "TC"

Sobre a função de impacto para embarcações offshore

Ao criar uma função de impacto para embarcações offshore, considere:

  • Embarcações são projetadas para resistir a condições marinhas adversas
  • Diferentes tipos de embarcações têm diferentes limiares de resistência
  • Muitas embarcações podem se deslocar para evitar o pior de uma tempestade
  • Os danos podem ser tanto estruturais quanto operacionais (interrupção de operações)
